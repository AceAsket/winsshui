# WinSSH UI

WinSSH UI — самостоятельный менеджер SSH-подключений для Windows, полностью
написанный на Python. Интерфейс построен на PySide6, локальный каталог — на
стандартном `sqlite3`, а сессии открываются в Windows Terminal через `wt.exe`.

## Возможности

- импорт конкретных `Host`-алиасов из `%USERPROFILE%\.ssh\config`;
- рекурсивная поддержка `Include` с отображением и редактированием исходного файла;
- создание, редактирование, клонирование и удаление подключений;
- расширенные настройки `ProxyJump`, keepalive, agent forwarding, compression,
  `RemoteCommand`, `RequestTTY` и SSH-туннелей;
- импорт из WinSCP, PuTTY, MTPuTTY, SuperPuTTY, FileZilla SFTP и mRemoteNG;
- получение эффективных параметров через `ssh -G`;
- поиск по алиасу, адресу, пользователю, группе, тегам и заметкам;
- виртуальная папка «Избранное» и пользовательские группы;
- пользовательские теги и многострочные заметки для каждого подключения;
- древовидные папки с вложенными путями `Source/Folder/Subfolder`;
- запоминание раскрытых и свёрнутых папок между запусками и обновлениями;
- назначаемые иконки устройств (сервер, роутер, коммутатор, firewall, NAS,
  облако, рабочая станция, база данных и терминал) с автоопределением;
- история последних 100 запусков;
- открытие SSH в новой вкладке или правой split-панели Windows Terminal;
- экспериментальное встраивание настоящего окна Windows Terminal внутрь приложения
  с его вкладками, изменением размера и безопасным отделением обратно на рабочий стол;
- сохранённые рабочие пространства из нескольких вкладок и split-панелей;
- диагностика подключения и `ssh-agent` без запроса пароля;
- менеджер ключей из `~/.ssh`: fingerprint, состояние `ssh-agent`, создание
  ED25519/RSA, добавление и удаление из agent, копирование публичного ключа;
- установка выбранного публичного ключа в `authorized_keys` через stdin с последующей проверкой;
- глобальные и привязанные к хосту командные сниппеты с запуском через `ssh.exe`;
- контекстное меню подключения, двойной клик для запуска и копирование SSH-команды;
- экспорт и восстановление ZIP-копии файлов настроек и локального каталога;
- журнал событий и экспорт диагностического ZIP без конфигурации, ключей и паролей;
- хранение выбранных SSH-паролей в Windows Credential Manager через отдельный askpass;
- ручная и ежедневная проверка новых версий, загрузка installer и проверка SHA-256;
- русские подписи интерфейса и перевод стандартных диалогов Qt;
- менеджер Local/Remote/Dynamic-туннелей с проверкой занятых портов, опциональным
  запуском вместе с приложением и автоперезапуском;
- системный трей с избранными, недавними подключениями и быстрым доступом к туннелям;
- палитра быстрого запуска `Ctrl+K` с действиями Terminal, split, SFTP/SCP, WinSCP и туннелей;
- встроенный SFTP-браузер и загрузка/скачивание файлов и каталогов через штатный `scp.exe`;
- массовые действия по папке или произвольному набору подключений;
- параллельный монитор SSH: задержка, причина ошибки и последний успешный вход;
- расширенные рабочие пространства: именованное окно, направление и размер split,
  заголовок и цвет вкладки;
- восстановление экземпляров подключений предыдущего сеанса с подтверждением при
  запуске; уже работающие `ssh.exe` повторно не открываются;
- безопасная повторная синхронизация импортированных подключений с обнаружением конфликтов;
- открытие выбранного подключения в WinSCP без передачи пароля;
- безопасный помощник для ошибки `REMOTE HOST IDENTIFICATION HAS CHANGED`: просмотр
  сохранённого fingerprint, подтверждаемое удаление старой записи и резервная копия `known_hosts`;
- передача аргументов процессам списком, без PowerShell/cmd и конкатенации строк.

OpenSSH остаётся источником истины: приложение запускает `ssh.exe <alias>`, а
разрешение `Include`, wildcard-блоков и системных настроек выполняет сам SSH.
Приватные ключи приложение не хранит. Пароль можно сохранить только явным
действием пользователя: он помещается в Windows Credential Manager и никогда не
записывается в SSH config, SQLite, резервную копию, аргументы процесса или журнал.
Резервная копия также намеренно не включает приватные ключи и `known_hosts`.

### Встроенный Windows Terminal

По умолчанию WinSSH UI создаёт отдельное именованное окно Windows Terminal и
размещает его на странице **«Терминал»** внутри приложения. Новые подключения,
рабочие пространства и командные сниппеты направляются в его вкладки и split-панели.
Кнопка **«Открыть отдельно»** возвращает окно Terminal на рабочий стол без
завершения SSH-процессов. При настоящем выходе из WinSSH UI встроенное окно
закрывается вместе с его SSH-подключениями; скрытие приложения в системный трей
не считается выходом и оставляет терминал работающим.

Это экспериментальный Win32-режим: Windows Terminal не предоставляет публичный
Python API для встраивания установленного приложения. Если Windows запрещает
межпроцессное переподчинение окна либо окно не удаётся обнаружить, оно остаётся
обычным отдельным окном. Режим можно отключить через меню шестерёнки.

### Сохранённые пароли

Кнопка **«Сохранить пароль»** создаёт Generic Credential, доступный текущей
учётной записи Windows. При запуске прямого SSH-подключения OpenSSH вызывает
отдельный `WinSSH-AskPass.exe`; в окружении передаётся только идентификатор
подключения. Askpass отвечает только на запрос `password` и не подтверждает ключ
хоста или passphrase приватного ключа. Для `ProxyJump`, рабочих пространств и
фоновых туннелей по-прежнему рекомендуется SSH-ключ и `ssh-agent`.

### Смена ключа сервера

Кнопка **«Ключ хоста…»** в карточке подключения показывает запись из `known_hosts`, включая
SHA-256 fingerprint. Если сервер был переустановлен и смена ключа подтверждена администратором,
старую запись можно удалить. Перед изменением создаётся копия вида
`known_hosts.winsshui-<дата>-<время>.bak`. Новый ключ никогда не принимается автоматически:
при следующем подключении его fingerprint необходимо сравнить с доверенным значением.

## Создание и импорт

Кнопка **«Новое подключение»** создаёт отдельный `Host`-блок в OpenSSH config.
Перед изменением существующего файла создаётся резервная копия `config.bak`, а
запись выполняется атомарно.

Редактор разделён на вкладки «Основное», «Сеть», «Туннели» и «Запуск». При
редактировании или удалении общий блок вида `Host one two` обрабатывается без
удаления остальных алиасов.

Кнопка **«Импорт»** автоматически сканирует:

- PuTTY и WinSCP в `HKCU` Registry;
- стандартные расположения WinSCP INI, MTPuTTY и SuperPuTTY XML;
- FileZilla `sitemanager.xml` — только SFTP;
- незашифрованные SSH-узлы mRemoteNG.

Можно также выбрать произвольный `.ini`, `.xml` или `.config` файл. Перед
сохранением показывается таблица с флажками и редактируемой папкой назначения.
Можно массово назначить папку всем отмеченным строкам. Автоимпорт всегда
подписывает источник: `PuTTY`, `WinSCP`, `SuperPuTTY/<исходная папка>` и т. п.
Пароли игнорируются. Ключи `.ppk` нужно предварительно конвертировать в формат
OpenSSH через PuTTYgen.

Кнопка **«Синхронизировать импорт»** повторно сканирует известные источники.
Приложение обновляет подключение только тогда, когда локальный `Host` не менялся
после предыдущего импорта. Конфликты пропускаются, а перед пакетными изменениями
создаётся `config.winsshui-sync.bak`. Удалённые из внешнего клиента подключения
автоматически не удаляются.

## Диагностика, туннели и WinSCP

Проверка подключения использует `BatchMode=yes`: пароль не запрашивается и не
сохраняется. Результат различает сетевые ошибки, проблемы host key и отсутствие
подходящего ключа. Состояние агента определяется через `ssh-add -l`.

Управляемые туннели запускаются через `ssh -N -T` с `ExitOnForwardFailure=yes`.
Для фонового подключения нужен ключ или `ssh-agent`; неизвестный или изменённый
host key автоматически не принимается. Перед запуском локальных пробросов
проверяется доступность listen-портов. Автозапуск и автоматический перезапуск
включаются отдельно для каждого подключения в **«SSH-инструменты → Менеджер туннелей»**.
Менеджер также показывает локальные адреса, uptime, число рестартов и журнал;
адрес можно скопировать или открыть как HTTP URL.

Кнопка **«Установить ключ»** передаёт публичный ключ серверу через стандартный
ввод `ssh.exe`, идемпотентно добавляет его в `authorized_keys` и затем проверяет
выбранным приватным ключом в режиме `BatchMode=yes`. Ключ хоста всегда проверяется строго.

Значок в системном трее содержит избранные и недавние подключения. Режим
**«⚙ → Закрывать окно в системный трей»** выключен по умолчанию.

## SFTP, SCP и WinSCP

Кнопка **«Файлы SFTP / SCP»** открывает удалённый каталог через штатный
`sftp.exe`, передавая навигационные команды через стандартный ввод. Файлы и каталоги передаются отдельным `scp.exe`; команды запускаются
списком аргументов и сохраняют строгую проверку host key. Для фоновой работы
рекомендуются ключ и `ssh-agent`, сохранённый пароль может быть передан только
через WinSSH AskPass. Текущий удалённый путь сохраняется в локальном каталоге и
используется при следующем открытии SFTP или WinSCP.

Если на роутере или embedded-системе отсутствует `sftp-server`, приложение
автоматически переключает просмотр каталога на `ssh + ls`, а передачу — на
классический протокол SCP. Режим запоминается до закрытия файлового окна.

Если WinSCP установлен, кнопка **«Открыть в WinSCP»** передаёт SFTP-адрес,
пользователя, порт и путь к ключу. Пароль в URL или аргументы процесса не попадает.

## Запуск из исходников

Требуются Python 3.10+, Windows Terminal и OpenSSH Client.

```powershell
python -m venv .venv
.\.venv\Scripts\python.exe -m pip install -e .
.\.venv\Scripts\python.exe -m winsshui
```

## Тесты

```powershell
.\.venv\Scripts\python.exe -m unittest discover -s tests\python -v
```

## Сборка EXE

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements-build.txt
.\.venv\Scripts\python.exe build.py
```

Результаты создаются в `dist\WinSSH-UI.exe`, `dist\WinSSH-AskPass.exe` и, если
установлен Inno Setup, `dist\WinSSH-UI-Setup.exe`. Python на целевом компьютере
не требуется. При первом запуске распаковка Qt во временный каталог может занять
несколько секунд.

Готовые установщики и portable-сборки публикуются на странице
[GitHub Releases](https://github.com/AceAsket/winsshui/releases/latest). Приложение
проверяет последний релиз не чаще одного раза в сутки; ручная проверка находится
в меню **«Данные → Проверить обновления…»**. Установщик загружается во временный
атомарный файл, сверяется с SHA-256 digest GitHub и запускается только после
отдельного подтверждения пользователя.

Установщик Inno Setup работает без прав администратора и устанавливает основное
приложение вместе с `WinSSH-AskPass.exe` в профиль текущего пользователя.

## Выпуск новой версии

1. Указать одинаковую версию в `pyproject.toml` и `src/winsshui/__init__.py`.
2. Закоммитить и отправить изменения в GitHub.
3. Создать и отправить тег вида `v0.8.0`.

Workflow `.github/workflows/release.yml` на Windows запускает тесты, проверяет
соответствие тега версии, собирает основной EXE, askpass helper и Inno Setup,
формирует `SHA256SUMS.txt` и создаёт GitHub Release. Ручной запуск workflow создаёт
только проверочный artifact и не публикует релиз без тега.

## Данные

- SSH-конфигурация: `%USERPROFILE%\.ssh\config`;
- метаданные приложения: `%LOCALAPPDATA%\WinSshUi\catalog.db`.
- журнал: `%LOCALAPPDATA%\WinSshUi\logs\winsshui.log`.

## Структура

- `src/winsshui` — всё приложение;
- `tests/python` — unit- и smoke-тесты;
- `winsshui.spec` — воспроизводимая конфигурация PyInstaller;
- `docs/architecture.md` — архитектурные границы и развитие.
