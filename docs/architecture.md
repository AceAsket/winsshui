# Архитектура WinSSH UI

Проект полностью реализован на Python. В рабочем дереве нет смешения с C# или
.NET-компонентами.

```text
PySide6 UI
  ├─ SshConfigReader → ~/.ssh/config → recursive Include/glob
  ├─ SshConfigWriter → atomic ~/.ssh/config update + config.bak
  ├─ WindowsClientImporter → Registry / INI / XML
  ├─ SshConfigurationResolver → ssh.exe -G
  ├─ SshDiagnostics → ssh.exe BatchMode + ssh-add -l
  ├─ KnownHostsManager → ssh-keygen -F/-R + known_hosts backup
  ├─ SshKeyManager → ~/.ssh + ssh-keygen/ssh-add
  ├─ PublicKeyInstallCommand → ssh.exe stdin → authorized_keys
  ├─ OpenSshTransferManager → sftp.exe batch / scp.exe
  ├─ ConnectionHealthDialog → parallel QProcess → persisted health snapshot
  ├─ CommandSnippet → sqlite3 → wt.exe → ssh.exe -t
  ├─ BackupManager → ZIP(manifest + configs + SQLite snapshot)
  ├─ Update downloader → GitHub Releases API → QSaveFile + SHA-256
  ├─ WindowsCredentialStore → CredWriteW/CredReadW → WinSSH-AskPass.exe
  ├─ Rotating log → redaction → diagnostics ZIP
  ├─ TunnelManager → port probe + ssh.exe -N -T + opt-in restart
  ├─ ConnectionCatalog → sqlite3
  ├─ WindowsTerminalLauncher → wt.exe → ssh.exe <alias>
  └─ WinScpLauncher → WinSCP.exe → sftp://...
```

## Компоненты

### `SshConfigReader`

Извлекает конкретные алиасы из директив `Host`. Wildcard и отрицательные
паттерны не становятся отдельными элементами списка. Извлечённые поля нужны для
быстрого отображения, но не заменяют интерпретацию OpenSSH. Директивы `Include`
раскрываются рекурсивно с поддержкой glob и защитой от циклов. Каждый Host
сохраняет `source_path`, поэтому изменение направляется в исходный файл блока.

### `BackupManager`

Создаёт ZIP с manifest, файлами настроек внутри `~/.ssh` и согласованным снимком
SQLite через Backup API. Приватные ключи и `known_hosts` не экспортируются.
Перед восстановлением каждого существующего файла создаётся копия
`*.before-restore-<дата>.bak`; пути проверяются от выхода за `~/.ssh`.

### `SshConfigurationResolver`

Запускает `ssh.exe -G -- <alias>` и показывает эффективные `hostname`, `user`,
`port`, `identityfile` и `proxyjump`. PySide6 использует `QProcess`, поэтому UI
не блокируется; запрос прерывается через пять секунд или при смене хоста.

### `SshConfigWriter`

Проверяет алиас, адрес, порт, сетевые параметры и правила туннелей, формирует
OpenSSH `Host`-блок, создаёт `config.bak` и заменяет конфигурацию атомарно через
временный файл в том же каталоге. Поддерживает добавление, редактирование и
удаление. При изменении одного алиаса из общего `Host a b` выделяет его в
отдельный блок и сохраняет настройки оставшихся алиасов.

### `WindowsClientImporter`

Объединяет read-only импортёры PuTTY/WinSCP Registry, WinSCP INI, MTPuTTY и
SuperPuTTY XML, FileZilla SFTP и незашифрованного mRemoteNG XML. Все форматы
нормализуются в одну модель предпросмотра. Поля паролей не читаются и не
переносятся; `.ppk` отмечается как требующий конвертации. Папка назначения
всегда получает корень с названием источника (`SuperPuTTY/…`, `WinSCP/…`), но
может быть изменена для одной или нескольких строк до импорта.

### Дерево подключений

`group_name` хранит путь с разделителем `/`. UI строит из него вложенные узлы,
поэтому импортированные подключения сразу разбиваются по источникам и исходным
каталогам клиентов, а не отображаются единым плоским списком. Выбранная иконка
устройства хранится рядом с группой в SQLite; если она не задана, UI определяет
тип по алиасу, адресу и папке, не изменяя OpenSSH config. Избранные подключения
дополнительно отображаются ссылками в виртуальной верхней папке «Избранное»,
оставаясь при этом в своих обычных группах.

### `ConnectionCatalog`

Хранит избранное, группы, теги, заметки, настройки туннелей, состояние раскрытия
папок, последние 100 запусков, рабочие пространства и
происхождение импорта: тип источника, идентификатор, fingerprint данных и время
синхронизации. Схема мигрирует старую базу добавлением nullable-полей. Запросы
параметризованы, SQLite работает в WAL-режиме, а соединения гарантированно
закрываются после каждой короткой транзакции.

### Пароли

SQLite, SSH config, резервные ZIP, журналы и аргументы процессов не содержат
пароли. Секрет хранится как Generic Credential в Windows Credential Manager под
хешированным идентификатором алиаса. `WinSSH-AskPass.exe` читает секрет в отдельном
короткоживущем процессе; основное приложение передаёт OpenSSH только идентификатор
через окружение и включает `SSH_ASKPASS_REQUIRE=force`. Helper отказывается
отвечать на всё, кроме password prompt, поэтому не может автоматически принять
неизвестный ключ хоста. Для ProxyJump и фоновых процессов предпочтительны ключи
и `ssh-agent`.

### `WindowsTerminalLauncher`

Формирует список аргументов для `subprocess.Popen`, не используя командную
оболочку. Все сессии направляются в именованное окно Terminal `winsshui`.
Рабочие пространства формируются последовательностью `new-tab` и `split-pane`.

### Диагностика и туннели

Диагностика запускает SSH с `BatchMode=yes`, ограниченным `ConnectTimeout` и без
интерактивного ввода, затем отдельно проверяет агент через `ssh-add -l`.
Управляемые туннели работают как дочерние `QProcess` с `-N -T`, строгой проверкой
host key и `ExitOnForwardFailure=yes`. Локальные listen-порты проверяются до
старта. Автозапуск и экспоненциальный перезапуск хранятся как явные настройки
отдельного подключения; при закрытии приложения процессы завершаются.

### Синхронизация импорта

Нормализованный fingerprint содержит адрес, пользователя, порт и ключ. При
повторном сканировании источник обновляется только если текущий локальный
fingerprint совпадает с последним применённым. Одновременное изменение источника
и локального Host считается конфликтом и не перезаписывается.

### `WinScpLauncher`

Ищет WinSCP через `PATH`, Windows App Paths и стандартные каталоги. Запускает
графический клиент аргументами процесса с `sftp://` URL без пароля. Цепочка
`ProxyJump` пока не переносится автоматически, о чём UI предупреждает.

### `KnownHostsManager`

Ищет записи, в том числе с хешированными именами, штатной командой `ssh-keygen -F`. Для
нестандартного порта используется OpenSSH-форма `[hostname]:port`. Удаление выполняется только
после явного подтверждения пользователя через `ssh-keygen -R`; до запуска команды рядом с
`known_hosts` создаётся отдельная резервная копия. Приложение намеренно не добавляет результат
`ssh-keyscan` в доверенные ключи и не отключает `StrictHostKeyChecking`.

### Ключи и командные сниппеты

`SshKeyManager` перечисляет публичные части ключей и вычисляет fingerprint через
`ssh-keygen`. Создание ключа и добавление в agent открываются в Windows Terminal,
поэтому passphrase не читается приложением. Установка публичного ключа передаёт
его только через stdin SSH-процесса, не помещает в argv и после записи проверяет
`authorized_keys` неинтерактивным подключением с выбранной приватной частью.
Сниппеты хранятся в SQLite и
передаются удалённому `ssh.exe` отдельным аргументом без локального shell.

### Системный трей

`QSystemTrayIcon` строит меню при открытии, поэтому избранные и недавние элементы
всегда соответствуют каталогу. Закрытие в трей является пользовательской
настройкой и выключено по умолчанию; пункт «Выход» завершает дочерние туннели.

### Быстрый запуск и массовые действия

Палитра `Ctrl+K` фильтрует уже загруженную модель по алиасу, endpoint, папке,
тегам и заметке. Она не дублирует конфигурацию и передаёт выбранный объект тем же
методам запуска, что основное окно. Массовые изменения выполняют отдельные
короткие транзакции каталога; запуск туннелей разнесён таймерами.

### SFTP и SCP

Навигация выполняется одноразовым `sftp.exe`: путь передаётся через stdin,
а не добавляется к командной строке. Передача использует `scp.exe` с аргументами
без shell. Пароль не сохраняется и при наличии Credential Manager поступает в
OpenSSH через тот же ограниченный askpass helper. Неизвестный host key не
принимается автоматически.

### Рабочие пространства и health

Элементы рабочего пространства хранят направление и долю split, заголовок и
цвет; пространство хранит имя окна Windows Terminal. Старые записи мигрируют со
значениями `Vertical`, `0.5` и `winsshui`. Health-проверки выполняются до четырёх
параллельно в `BatchMode`, сохраняют только статус, задержку и время — без вывода
SSH и секретов.

### Упаковка

PyInstaller собирает PySide6, Python runtime и приложение в один оконный EXE.
Windows Terminal и OpenSSH намеренно не встраиваются и должны присутствовать в
системе.

### Обновления и релизы

Проверка обновлений асинхронно читает публичный endpoint последнего GitHub
Release через `QNetworkAccessManager`, сравнивает числовую версию и сохраняет
только время проверки в SQLite. Installer загружается в `QSaveFile`, одновременно
хешируется и публикуется по конечному пути только при совпадении SHA-256 с digest
asset из GitHub API. Запуск требует отдельного подтверждения пользователя.

Тег `vX.Y.Z` запускает Windows workflow с тестами, двумя PyInstaller-сборками и
Inno Setup. Workflow проверяет совпадение тега, версии пакета и
`winsshui.__version__`, затем прикладывает installer, portable EXE, askpass helper
и SHA-256 checksums. PR и push в `main` дополнительно проверяются отдельным CI.
